# 校园跑腿系统 Campus Errand System

基于微信小程序的校园跑腿服务平台，为在校师生提供便捷的跑腿代办服务。

## 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | 微信小程序（WXML + WXSS + JavaScript） |
| 后端 | Spring Boot 2.x |
| 数据库 | MySQL 8.0 |
| 接口规范 | RESTful API |

## 核心功能

- **用户下单** — 发布跑腿任务，填写取送地址、物品描述、酬金等信息
- **跑腿员接单** — 浏览可接任务，抢单/接单，实时查看任务详情
- **订单管理** — 订单状态流转（待接单 → 进行中 → 已完成），支持取消与评价
- **用户中心** — 注册登录、个人信息管理、历史订单查询

## 系统架构

本项目采用**前后端分离架构**：

```
┌─────────────┐       HTTPS/JSON        ┌──────────────┐       JDBC        ┌─────────┐
│  微信小程序  │  ◄──────────────────►  │  Spring Boot  │  ◄────────────►  │  MySQL  │
│  （前端）    │                         │  （后端API）   │                   │         │
└─────────────┘                         └──────────────┘                   └─────────┘
```

- **前端**：微信小程序负责界面展示与用户交互，通过 wx.request 调用后端接口
- **后端**：Spring Boot 提供 RESTful API，处理业务逻辑与数据持久化
- **数据库**：MySQL 存储用户、订单、任务等核心业务数据
